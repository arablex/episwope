# OSINT Shadow Engine — журнал валидации

Приватный режим наблюдения. Цель: проверить **правильно ли мы интерпретируем косвенные сигналы**.
Метод: фиксируем что движок предсказал → ждём → сверяем со свершившимся (WHO / новости / официал).

> Этот файл я (Claude) обновляю между сессиями. Браузерный журнал копится отдельно
> в `localStorage.vigilo_osint_journal` — экспорт командой `osintJournal()` в консоли.

## Tier'ы движка
- `covert_elevated` — официально тихо, поведенчески громко (главная гипотеза moat'а)
- `elevated_watch` — повышенное наблюдение
- `nominal` — норма

## Формула (детерминированная)
`behavioral = 0.30·conflict + 0.20·unrest + 0.22·infra + 0.16·currency + 0.12·border` (0–5)
`divergence = behavioral − officialActivity`
Covert если: `behavioral ≥ 3.5  AND  official ≤ 1.0  AND  divergence ≥ 2.5`

## Что проверяем в динамике
1. **Ложные срабатывания** — covert там где спокойно → формула слишком чувствительна.
2. **Пропуски** — реальный кризис случился, движок молчал → не хватает сигнала/веса.
3. **Lead time** — за сколько часов/дней до официала движок поднял флаг.
4. **Вес currency** — не переоцениваем ли хронически слабые валюты (Венесуэла годами −99% — это фон, не «скрытый кризис») → возможно нужен **delta**, а не абсолют.

## Открытые вопросы по интерпретации
- Хроническая нестабильность ≠ острый скрытый кризис. Нужен признак **резкого изменения**, не уровня.
- «Официально тихо» для закрытых стран (Туркменистан, КНДР) — норма, не аномалия. Не штрафовать за непрозрачность саму по себе.
- Интернет-шатдаун в Индии (Кашмир) — рутина в одном регионе, не кризис страны целиком. Нужна гео-локализация.

- **ФИКС #3 / ICEWS-промпт разобран. Применён INFORM-демпфер.**

  Из 4-step промпта (DARPA ICEWS) принят 1 кусок, 3 отклонены честно:
  - ✅ **INFORM Hazard×Vuln/Coping** — построено (бесплатно, верно, лифтит каждый сигнал)
  - ⚠️ Каскад climate→food→unrest — спецификация как детерминированный FSM «hypothesis tracker», НЕ ICEWS-ML (не строим пока)
  - ❌ Flight ticket-asymmetry/pricing — несорсится (повтор нашего общего решения по FR24/OpenSky)
  - ❌ «85% вероятность» — фабрикация (журнал n=0, нет ground truth)
  - ❌ PL/pgSQL compute в БД — против архитектуры (движок один в JS, паритет-тест)

  `country-structural.json` (INFORM 2024, статичный seed, 24 страны) + `informModifier()` в обоих движках (сервер + браузер, формула идентична).
  Формула: `F=clamp((vuln·coping)/100,0,1); M=clamp(0.70+F·1.6, 0.70,1.60); behEff=min(5, behRaw·M)`.

  Тест (одинаковый сырой сигнал 3.05):

  | Страна | M | raw | eff | tier |
  |--------|--:|----:|----:|------|
  | Судан (фрагильн.) | 1.60 | 3.05 | 4.87 | **covert_elevated** |
  | Германия (устойч.) | 0.73 | 3.05 | 2.22 | nominal |
  | нет данных INFORM | 1.00 | 3.05 | 3.05 | elevated_watch (fallback нейтрален) |

  Вывод: идентичный сигнал теперь корректно опаснее в структурно хрупкой стране. Закрыт пробел «сигнал в Швейцарии = сигнал в Южном Судане». Паритет сервер↔браузер сохранён (формула скопирована точно). Fallback M=1.0 — никогда не ломается.

- **Решение по секвенированию (сам определил):** Фазу 3 НЕ строить сейчас. Причина — мой же аргумент: Фазы 1–2 + INFORM не задеплоены, журнал n=0, dispatch-очередь пуста. Строить human-review поверх пустоты = «5-й этаж до заселения 1-го». Правильно: **задеплоить → дать журналу копиться 2–4 недели → вернуться к Фазе 3 когда есть реальные данные и есть что гейтить.**

- **Покрытие стран — фикс «пустых карточек».**

  Проблема: открываешь страну без активных событий → карточка выглядит сломанной.
  Корень: event-данные (конфликт/беспорядки) есть только где есть события — это **корректно**, не баг. Лечится не докруткой событий, а структурным baseline.

  Сделано:
  1. **Пустая карточка → структурный профиль.** Нет live-индекса → показываем INFORM (Hazard/Vulnerability/Coping) + уверенное «Активных угроз не зафиксировано · нормальное состояние». Никогда не выглядит сломанным.
  2. **INFORM-seed расширен 24 → 69 стран**: вся G20 + все INFORM very-high/high-risk + ключевые туристические/транзитные. Это набор, который реально открывают.
  3. Длинный хвост (остальные ~120) → fallback M=1.0 (нейтрально) + graceful «угроз нет». Не ломается.

  Честная граница: полное покрытие всех ~191 стран = ингест официального годового датасета INFORM (CSV). Руками 191 строку не фабрикую (нарушило бы дисциплину «без фейк-данных»). 69 покрывают практически весь реальный трафик открытий; хвост безопасно деградирует.

  Что глобально собрать НЕЛЬЗЯ честно: wastewater (программы не везде), power-grid (точечно). Остаются страновыми, показываются как «нет сигнала», не «сломано».

- **ПЕТЛЯ ЗАМКНУТА: авто-разметчик исходов построен.**

  `_lib/osint-label.mjs` + воркер: каждый прогноз через 7–30 дней сверяется с официальной реальностью. Это вторая половина петли (была только первая — запись прогнозов).

  Принципы (честные):
  - **Независимость метки**: подтверждение по официальной эскалации (risk_index composite/conflict/health), НЕ по тому же indirect-фиду (валюта/интернет), что дал прогноз. Без само-подтверждения.
  - **Opaque-bias защита**: закрытые страны (T≤0.45) без подтверждения → `opacity_unverifiable`, НЕ miss. Не наказываем самые ценные прогнозы.
  - **Recall-сторона**: `detectMissed()` ловит официальные эскалации, которые мы НЕ флагнули → видимость false-negative. Precision без recall врёт.
  - Словарь честный: `not_confirmed` ≠ «ошиблись».

  Тест: SD→confirmed(lead 10д), DE→not_confirmed, TM→opacity_unverifiable, IR→pending, KE→missed. Всё корректно.

  Scorecard в `/api/admin/osint-journal`: precision, avg_lead_days, opacity_unverifiable, missed_total + честная плашка «n<20 → только направленно, не статистически» (защита от переобучения на малом n).
